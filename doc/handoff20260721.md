## What changed today

### 1. Bug fix: canvas not rendering

**Symptom** (reported by Chatri): opening `/issue-graph`, submitting an issue ID showed no graph — the canvas area was blank.

**Root cause**: Nuxt's component auto-import registers `app/components/issuegraph/*.vue` files with a folder-name prefix *unless* the filename exactly matches the folder name. So:
- `app/components/issuegraph/IssueGraphCanvas.vue` is actually registered as `<IssuegraphIssueGraphCanvas>`, not `<IssueGraphCanvas>`.
- `app/components/issuegraph/IssueNode.vue` is actually registered as `<IssuegraphIssueNode>`, not `<IssueNode>`.

The templates referenced the unprefixed names, so Vue silently failed to resolve the components and nothing rendered. Confirmed via the generated `.nuxt/components.d.ts`.

**Fix**:
- `app/pages/issue-graph/index.vue` — `<IssueGraphCanvas>` → `<IssuegraphIssueGraphCanvas>`.
- `app/components/issuegraph/IssueGraphCanvas.vue` — `<IssueNode>` → `<IssuegraphIssueNode>` (also added explicit `(id: number)` param type on the inline expand handler to clear a resulting TS7006 diagnostic).

### 2. UI polish: long issue subjects now wrap instead of truncating to one line

**Ask** (Chatri): long subjects on graph nodes were cut to a single truncated line — wanted multi-line display instead.

- `app/components/issuegraph/IssueNode.vue`:
  - Subject `<div>`: `truncate` → `line-clamp-3 wrap-break-word` (shows up to 3 lines, wraps long words; `title="issue.subject"` tooltip kept for the full text beyond that).
  - Node root `<div>`: added fixed `w-55` (220px) so wrapping is predictable and matches the width dagre lays out against.
- `app/composables/useDagreLayout.ts`: `NODE_HEIGHT` `80` → `110` so dagre reserves enough vertical space for a 3-line subject and nodes don't visually overlap.
