# Handoff — 2026-07-20 — Issue Relation Graph Visualization

## Goal

Add a page where a user enters a Redmine issue ID (e.g. `334586`) and sees that issue plus its related issues (Redmine relations: `relates`/`blocks`/`duplicates`/`precedes`/`copied_to`/`copied_from`, plus parent/child hierarchy, across projects) rendered as an interactive node graph, using [Vue Flow](https://vueflow.dev/).

## What was built

### New dependencies (`package.json` / `bun.lock`)
`@vue-flow/core`, `@vue-flow/background`, `@vue-flow/controls`, `@dagrejs/dagre` (bundled TS types, no `@types/dagre` needed).

### Shared types
- `shared/types/IssueGraph.d.ts` (new) — `IssueGraphNode`, `IssueGraphEdge`, `IssueGraphResponse`, `IssueGraphOptions`, `IssueGraphEdgeType`.
- `shared/types/Issue.d.ts` (extended) — `RawIssue` gained optional `relations?: RawIssueRelation[]` and `children?: RawIssueChild[]`, populated only when Redmine is queried with `include=relations,children`.

### Backend
- `server/utils/issueGraph.ts` (new) — `fetchIssueGraph(rootId, params)`: bounded BFS over Redmine relations/parent/children starting from the root issue.
  - `visited: Set<number>` dedupes nodes; edges deduped by a stable id (Redmine relation `id` for relation edges, `parent-<parentId>-<childId>` for hierarchy edges).
  - An edge is only recorded once its other endpoint is guaranteed to be fetched (already visited or newly enqueued) — this avoids "dangling" edges pointing at issues that were cut off by depth/maxNodes/timeout. This was a bug caught and fixed by test-writing today (see Known Issues below — now fixed, not outstanding).
  - Three safety limits, all client-overridable but hard-capped server-side: `depth` (≤5), `maxNodes` (≤200), `timeoutMs` (≤30000) — see `clampGraphOptions()`.
  - Response includes `truncated: boolean` + `truncatedReason: 'maxNodes' | 'timeout' | 'depth'`, so the UI can distinguish "safety cap hit" (show a warning) from "just reached the requested depth" (expected, handled via per-node expand instead).
- `server/api/issues/graph.get.ts` (new) — `GET /api/issues/graph?id=&depth=&maxNodes=&timeoutMs=`. 404 if the root issue doesn't exist; 400 if `id` is missing/non-numeric; per-node fetch failures during traversal are logged and skipped, not fatal.

### Frontend
- `app/composables/useIssueGraph.ts` (new) — `getIssueGraph()` (initial load), `expandNode()` (single-node lazy expand, depth=1), `toVueFlowElements()` (maps API shape → Vue Flow `Node[]`/`Edge[]`), `relationEdgeStyle()` (color/dash per relation type), `mergeGraphResponse()` (dedupe-merge new nodes/edges into existing canvas state).
- `app/composables/useDagreLayout.ts` (new) — wraps `@dagrejs/dagre` to auto-position nodes (Vue Flow itself does no layout).
- `app/components/issuegraph/IssueNode.vue` (new) — custom Vue Flow node: issue id (linked to Redmine), subject, tracker/status badges, expand button (shown when `hasMoreRelations`).
- `app/components/issuegraph/IssueGraphCanvas.vue` (new) — `<VueFlow>` wrapper with `Background`/`Controls`, registers the `issue` node type via the `#node-issue` slot.
- `app/pages/issue-graph/index.vue` (new) — issue ID input + collapsible "Advanced Options" (depth/maxNodes/timeout), truncation warning banner, loading state, toast on error.
- `app/components/sidebar/sidebarItems.ts` — added "Issue Graph" nav entry → `/issue-graph`.

### Tests
- `server/utils/issueGraph.test.ts` (9 tests) — mocks `axios`, covers depth cutoff, full traversal, symmetric-relation dedup, maxNodes cap (asserts no dangling edges), root-not-found rethrow, mid-traversal 404 tolerance, `clampGraphOptions` defaults/ceiling/floor clamping.
- `app/composables/useIssueGraph.test.ts` (4 tests) — `toVueFlowElements`, `relationEdgeStyle` (incl. unknown-type fallback), `mergeGraphResponse` dedup of nodes and edges.
